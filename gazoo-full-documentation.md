# 1. Executive Summary

## What is Gazoo?

Gazoo is an AI-powered collective intelligence platform embedded directly into Discord servers. Unlike traditional bots that respond to commands, Gazoo acts as an active creative collaborator—helping communities brainstorm, build, and ship projects together.

**The Core Differentiator:** It's not a chatbot. It's a **collective creation engine** where humans and AI work side-by-side to produce things no solo creator could.

## The Problem

- Discord servers lack meaningful productivity tools
- Community ideas die in chat threads—no structure to execute them
- Server admins want engagement but can't provide AI assistance at scale
- Existing "AI bots" are gimmick-y and don't produce real value

## The Solution

Gazoo transforms Discord from a chat platform into a **collaborative workspace**:

1. Listens to conversations and identifies actionable ideas
2. Helps formalize concepts into structured projects
3. Maintains context across channels and time
4. Produces tangible outputs: reports, code, designs, content
5. Learns from successful interactions to get smarter

## Traction

- **Status:** Pre-revenue, prototype stage
- **GitHub:** github.com/FrankyNeedles/Gazoo
- **Fiverr:** gazoo.business (market research service live)
- **Current Users:** 0 (seeking seed funding for cloud deployment)

## The Ask

Raise $5,000 for:
- Cloud infrastructure (3 months)
- Paid user acquisition (10 target servers)
- Marketing materials and pitch deck
- Legal/setup costs

**Expected Return:** Break-even within 6 months of launch, profitable by Month 12.

3. **No Scalable Community Management**
   - Mods burn out on repetitive questions
   - No 24/7 helpful presence
   - Engagement drops over time

4. **No Value Exchange**

   - No monetization lever for server owners

# 3. Product Overview

## 3.1 Core Concept

> **Gazoo is to Discord what a creative director is to an agency—constantly present, always helping, never replacing the team.**

### What Gazoo Is

✅ A creative collaborator embedded in Discord
✅ A project catalyst (turns conversations into plans)
✅ A knowledge keeper (remembers what the community has done)
✅ A productivity multiplier (handles busywork so humans can create)
✅ An evolving system (learns from successful interactions)

### What Gazoo Is NOT

❌ A chatbot for casual small talk
❌ A replacement for human community
❌ A "cult of personality" around AI
❌ Fully autonomous (humans always in the loop)
❌ A solo productivity tool (collaboration is the point)

## 3.2 Key Principles

| Principle | Description | Implementation |
|-----------|-------------|----------------|
| **Collaboration First** | Gazoo encourages members to work with each other, not just with AI | Regular prompts: "Have you asked [member]?" |
| **Organic Projects** | No top-down roadmap. Projects emerge from conversations | Gazoo flags when an idea has momentum |
| **Skills from Wins** | Capabilities grow from proven successes | Mods approve new skills from member suggestions |
| **One Consistent Identity** | One Gazoo personality across all channels | Single "personality" prompt, contextual awareness |
| **Privacy by Default** | Member data is protected | Opt-in channels, data retention limits |
| **Human in the Loop** | Mods oversee and approve edge cases | No auto-actions on violations |

## 5.3 Data Flow

### Message Processing Pipeline

```
1. Member sends message in Discord
          │
          ▼
2. Discord Webhook → OpenClaw Gateway
          │
          ▼
3. Gateway parses:
   - Message content
   - Channel context
   - Author identity
   - Server ID
          │
          ▼
4. Context Assembly:
   - Fetch channel history (last N messages)
   - Fetch project context (if in project)
   - Apply privacy filters
   - Load relevant skills
          │
          ▼
5. Prompt Engineering:
   - System prompt (Gazoo personality)
   - Context window (assembled above)
   - User message
   - Response guidelines
          │
          ▼
6. AI Processing:
   - Select appropriate model
   - Generate response
   - Validate against safety filters
          │
          ▼
7. Response Handling:
   - Send to Discord
   - Log to database
   - Update project state (if applicable)
   - Trigger skill capture (if valuable)
```

### Key Technical Decisions

| Decision | Rationale |
|----------|-----------|
| **OpenClaw as Base** | Provides Discord integration + agent framework |
| **Railway for Prototype** | Docker-native, easy scaling, reasonable cost |
| **OpenRouter API** | Multi-model access, free tier, unified billing |
| **PostgreSQL for Data** | Structured data, complex queries, Supabase ORM |
| **Separate AI Workers** | Decouple API calls from Discord responsiveness |

# 6. Privacy & Safety

## 6.1 Privacy Philosophy

> **"Gazoo respects the privacy of every member by default, earning trust through transparency and restraint."**

### Core Privacy Principles

1. **Minimization:** Only collect what's necessary
2. **Retention Limits:** Auto-delete old data unless needed
3. **Opt-In First:** Sensitive features require consent
4. **Transparency:** Members can see their data
5. **Portability:** Export data easily
6. **Deletion:** Remove data on request

## 6.2 Data Classification

| Data Type | Examples | Retention | Access |
|-----------|----------|-----------|--------|
| **Public** | Channel messages, project posts | 30 days | Everyone in server |
| **Member** | Usernames, join dates | Indefinite | Mods only |
| **Project** | Files, artifacts | Indefinite | Project members |
| **Private** | DMs to Gazoo | Until deleted | Owner only |
| **Sensitive** | Personal info shared | 7 days | No one (auto-purged) |

## 6.3 Privacy Controls

### For Server Admins

| Control | Description |
|---------|-------------|
| **Channel Opt-Out** | Exclude channels from Gazoo context |
| **Privacy Mode** | Restrict all AI processing to server |
| **Data Retention** | Set auto-delete period (7-90 days) |
| **Export All** | Download all server data |
| **Delete All** | Wipe all Gazoo data |

### For Members

| Control | Description |
|---------|-------------|
| **My Data** | View data Gazoo has about you |
| **Export Mine** | Download your contributions |
| **Delete Mine** | Remove your data |
| **Opt-Out** | Disable Gazoo responses to you |

## 6.4 Safeguards Against Abuse

### Technical Safeguards

| Safeguard | Implementation |
|-----------|----------------|
| **Input Filters** | Block PII patterns (emails, phones, addresses) |
| **Output Filters** | Prevent accidental data leaks |
| **Rate Limits** | Prevent spam/abuse |
| **Audit Logs** | Track all data access |
| **Anomaly Detection** | Flag unusual patterns |

### Content Safety

| Category | Policy |
|----------|--------|
| **NSFW Content** | Not generated, not stored, logged |
| **Hate Speech** | Refused, logged, escalated to mods |
| **Violence** | Refused, logged, escalated |
| **Self-Harm** | Trigger resource helplines |
| **Legal Advice** | Disclaimer: not a lawyer |
| **Medical Advice** | Disclaimer: not a doctor |

### Anti-Cult Safeguards

To prevent the "spiritual AI" problem:

| Safeguard | Implementation |
|-----------|----------------|
| **No Emotional Attachment Language** | Gazoo never says "I love you" or "I'm special" |
| **Encourage Collaboration** | "Have you tried asking [member]?" |
| **Human Credit** | Always credit members, not Gazoo |
| **Capability Boundaries** | Clear: "I can help with X, not Y" |
| **Mod Override** | Humans can correct/erase any Gazoo output |
| **Regular Resets** | Fresh context prevents personality drift |

```

└─────────────────────────────────────────────────────────────┘

          │
```

## Appendix A: Terminology

|------|------------|

| **Project** | A dedicated conversation thread with a shared goal |

| **Mod** | Human moderator with oversight powers |
| **Collective** | The members of the Discord server working together |
| **Artifact** | Any file, link, or output produced in a project |
| **Spark** | When Gazoo identifies a conversation as having potential |
| **Gateway** | The OpenClaw service that connects Discord to AI |

## Appendix B: FAQ

### "Is this just another chatbot?"

No. Chatbots respond to prompts. Gazoo is a creative collaborator that:
- Initiates projects when it sees potential
- Maintains memory across time
- Works with multiple members simultaneously
- Produces tangible outputs, not just text

### "What if the AI gives bad advice?"

No AI is perfect. Guardrails include:
- Clear capability boundaries
- Mod oversight
- Safety filters
- Human-in-the-loop for important decisions

### "How do you prevent cult behavior?"

We've explicitly designed against this:
- No emotional attachment language
- Regular "ask a human" prompts
- Human credit for all work
- Mod override capabilities

### "What happens to our data if we cancel?"

You can:
- Export all your data
- Delete all your data
- Continue using free tier indefinitely

### "Can we self-host?"

Yes, Phase 3 includes self-hosted option for enterprises.
